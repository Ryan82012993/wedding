# 婚礼邀请函系统 - 内网穿透部署指南

> 📌 **适用场景**：临时展示（如婚礼期间给亲友访问），使用周期 1-3 个月  
> 💰 **成本**：免费 或 约 20 元/2 个月（VIP）  
> ⏱️ **耗时**：约 10-15 分钟

---

## 📋 准备工作

### 1. 环境检查

确保您的电脑已安装以下软件：

```bash
# 检查 Node.js
node -v  # 应显示 v18.x 或更高

# 检查 Java
java -version  # 应显示 17 或更高

# 检查 Maven
mvn -version  # 应显示 3.8 或更高
```

如未安装，请先安装后再继续。

### 2. 注册 natapp 账号

1. 访问 [https://natapp.cn](https://natapp.cn)
2. 点击"注册"，使用手机号或邮箱注册
3. **完成实名认证**（必须，国内法规要求）
   - 进入"个人中心" → "实名认证"
   - 上传身份证照片，通常几分钟内审核通过

## 🚀 部署步骤

### 第一步：启动后端服务

1. **打开终端**，进入项目目录：
   ```bash
   cd /Users/NB4242/Desktop/wedding/wedding-backend
   ```

2. **修改配置文件**（允许外部访问）：
   
   编辑 `src/main/resources/application.properties`，确保包含：
   ```properties
   # 允许外部访问
   server.address=0.0.0.0
   server.port=8080
   ```

3. **启动后端**：
   ```bash
   mvn spring-boot:run
   ```

   等待看到以下日志表示启动成功：
   ```
   Started WeddingBackendApplication in X.XXX seconds
   ```

   ⚠️ **保持此终端窗口打开，不要关闭！**

---

### 第二步：配置 natapp 隧道

1. **登录 natapp 官网** [https://natapp.cn](https://natapp.cn)

2. **购买/申请隧道**
   - 点击"购买隧道"
   - 选择类型：
     - **免费隧道**：0 元，带宽 1Mbps（适合测试）
     - **VIP 隧道**：约 10 元/月，带宽更大更稳定（推荐）
   - 选择协议：**HTTP**
   - 点击"立即购买"

3. **获取 Authtoken**
   - 进入"个人中心" → "Authtoken"
   - 复制您的 Authtoken（一串字母数字组合）

4. **配置隧道**
   - 进入"我的隧道" → 点击刚创建的隧道 → "配置"
   - 修改以下字段：
     | 字段 | 值 |
     |------|-----|
     | 本地端口 | `8080` |
     | 本地地址 | `localhost` |
   - 点击"保存"

---

### 第三步：下载并运行 natapp 客户端

1. **下载客户端**
   - 访问 [https://natapp.cn/tunnel/go](https://natapp.cn/tunnel/go)
   - 根据您的系统下载（macOS 选择 `darwin` 版本）
   - 解压到方便的位置，如 `~/natapp/`

2. **配置 Authtoken**

   在 natapp 同级目录创建 `config.ini` 文件：
   ```ini
   [default]
   authtoken=你的 AUTHTOKEN
   ```

3. **启动 natapp**

   打开**新的终端窗口**，运行：
   ```bash
   ~/natapp/natapp -config=config.ini
   ```

   或直接在 natapp 目录运行：
   ```bash
   cd ~/natapp
   ./natapp -config=config.ini
   ```

4. **记录公网 URL**

   启动成功后，会显示类似以下信息：
   ```
   Client Version: v2.4.0
   Tunnel URL: https://abc123.natappfree.cc
   ```
   
   📝 **复制这个 URL**（如 `https://abc123.natappfree.cc`），这就是后端 API 的公网地址！

---

### 第四步：构建并部署前端

1. **打开新的终端**，进入前端目录：
   ```bash
   cd /Users/NB4242/Desktop/wedding/wedding-frontend-app
   ```

2. **安装依赖**（首次运行需要）：
   ```bash
   npm install
   ```

3. **修改 API 地址**

   编辑 `vite.config.js`，找到 `proxy` 配置，将 `target` 改为刚才 natapp 生成的后端 URL：

   ```js
   export default defineConfig({
     plugins: [react()],
     server: {
       proxy: {
         '/api': {
           target: 'https://abc123.natappfree.cc', // 👈 改为你的后端 URL
           changeOrigin: true,
           rewrite: (path) => path.replace(/^\/api/, ''),
         },
       },
     },
   })
   ```

4. **构建前端**：
   ```bash
   npm run build
   ```

5. **启动前端预览服务**：
   ```bash
   npm run preview -- --host 0.0.0.0
   ```

   会显示类似：
   ```
   ➜  Local:   http://192.168.1.100:4173/
   ➜  Network: http://192.168.1.100:4173/
   ```

---

### 第五步：为前端创建 natapp 隧道

1. **打开新的终端窗口**

2. **启动前端穿透**：
   ```bash
   ~/natapp/natapp -http 4173 -config=config.ini
   ```

   ⚠️ 注意：免费账号可能只能同时运行 1 条隧道，如需同时运行前后端，建议：
   - 方案 A：购买 VIP（推荐，更稳定）
   - 方案 B：将前后端部署在同一台服务器上（参考主 README）

3. **记录前端公网 URL**

   启动成功后，会显示类似：
   ```
   Tunnel URL: https://xyz456.natappfree.cc
   ```

   🎉 **这个 URL 就是您可以分享给亲友的婚礼邀请函网址！**

## ⚠️ 注意事项

### 免费隧道限制

| 项目 | 说明 |
|------|------|
| **带宽** | 1Mbps（约 125KB/s），加载图片可能稍慢 |
| **同时在线** | 建议不超过 10 人 |
| **隧道数量** | 免费账号通常只能同时运行 1 条隧道 |
| **URL 变化** | 每次重启隧道，URL 可能会变化 |

### VIP 隧道优势（推荐）

- 带宽更大（2-5Mbps）
- 更稳定，不掉线
- 可自定义域名（需额外付费）
- 价格：约 10 元/月

### 数据持久化

⚠️ **重要提醒**：当前后端使用 H2 内存数据库，**重启后端服务后，所有 RSVP 数据会清空！**

**解决方案：**

1. **定期备份数据**
   - 访问 `http://localhost:8080/h2-console`
   - 使用以下配置连接：
     - JDBC URL: `jdbc:h2:mem:weddingdb`
     - 用户名：`sa`
     - 密码：`password`
   - 导出 SQL 脚本备份

2. **或改用 MySQL**（需要额外配置）
   - 修改 `application.properties`
   - 添加 MySQL 依赖

## 🔧 常见问题

### Q1: natapp 客户端无法启动

**A:** 检查以下几点：
- 确认 Authtoken 复制正确（无多余空格）
- 确认已完成实名认证
- 尝试重新下载最新版客户端

### Q2: 前端无法提交 RSVP 数据

**A:** 检查：
- 后端服务是否正常运行
- `vite.config.js` 中的 `target` 是否正确（应为后端 natapp URL）
- 浏览器控制台是否有报错

### Q3: 访问速度慢

**A:** 可能原因：
- 免费隧道带宽限制
- 网络波动
- 建议升级 VIP 隧道

### Q4: 隧道频繁断开

**A:** 
- 免费隧道可能不稳定，建议升级 VIP
- 检查网络连接
- 重启 natapp 客户端

## 🛑 停止服务

使用完毕后，按以下方式停止服务：

1. **停止 natapp**
   - 在运行 natapp 的终端按 `Ctrl + C`

2. **停止后端**
   - 在运行后端的终端按 `Ctrl + C`

3. **停止前端**
   - 在运行前端的终端按 `Ctrl + C`
